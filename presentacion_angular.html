Prototipado

  • Un objeto posee atributos y métodos. Un prototipo sólo posee argumentos.
  • La cantidad de argumentos pueden ser modificados por sí mismos en un prototipo (Polimorfismo)
  • Permite herencia múltiple y clonación, tanto de argumentos, como de prototipos
  • Un prototipo es siempre global (rootScope o contexto principal) dentro del módulo en el que se crea
  • De todos los conceptos de OOP se pueden sacar prototipos
  • No existe encapsulamiento de prototipos

¿súper heróico?

Angular.js permite...
  1. Mocking (prototipado rápido de UX)
  2. Auditabilidad E2E (End to End)
  3. Arquitecturas desacopladas
  4. Integración directa con HTTP
  5. AJAX Nativo
  6. Manipulación transversal del DOM ¡sin jQuery!
  7. Desarrollo de SPA (Single Page Applications)
  8. Acoplamiento con cualquier framework de desarrollo web
    (CodeIgniter, Yii, Django, Ruby on Rails, etc.)

Orientado a Front-end

100% Javascript

  1. No necesitas aprender otro lenguaje de programación
  2. HTML5 y CSS3 deseable (no limitativo)
  3. Aplicaciones híbridas
  4. Responsivo por defecto

De sintaxis específica (opinionated):

¡En Angular.js, TODO es un módulo!, y hay módulos...
  1. ... para manipular comportamientos (directivas)
  2. ... para compartir datos entre componentes (fábricas)
  3. ... para crear interfaces de acceso a componentes (servicios)
  4. ... para encapsular comportamientos (controladores)
  5. ... para separar acceso a datos (rutas)
  6. ... muchas cosas más.
